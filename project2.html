Banjongrat School Library Management System

Oct 2022 - May 2023

Link to Figma

Introduction

We consider adopting a computer system to assist with the library's book borrowing and returning processes, based on the proposals presented by the committee for addressing various issues.

The features of the book borrowing and returning system

  • Borrowing books
  • Returning books
  • Checking damaged books
  • Reporting a lost books
  • Payment of fines
  • Generating reports

    • Input Design with Figma

    1. Login screen

    Librarians or authorized personnel must log in before accessing the library management system. They need to enter their username, password, and click the 'Login' button. If users forget their password, they can click the 'Forgot Password' button to request a new one.

    2. Borrowing screen

    When users access the borrowing screen, no book list will be displayed until they enter their student ID to proceed with borrowing books.

    When users enter their student ID to proceed with borrowing books, the system will display the student's personal information and various details such as borrowing history and any outstanding fines.

    When students need to settle unpaid fines, users can generate a payment reminder by checking the box and clicking 'Print Payment Reminder' to initiate the payment process for outstanding fines. This action updates the status of the item to 'Printed'.

    When users wish to borrow a book, they should press the 'Borrow Book' button. This screen will display a list of all books available in the library. Users can select a book from the displayed list or search by book code.

    When users have selected the book they wish to borrow, clicking on the folder icon at the bottom will display a window showing the list of books they have borrowed.

    When users click 'Save' for the borrowing transaction, they will be directed to a summary screen displaying details such as the book code, book title, author, borrowing date, and due date.

    3. Return book screen

    When accessing the return book screen, no book list will be displayed until the user enters their student ID to proceed with returning books.

    When users enter their student ID to proceed with returning books, the system will display the student's personal information and various details such as borrowing history and any outstanding fines. After receiving the book from the student, users will press the 'Return Book' button and place the book into a basket for further inspection.

    After pressing the 'Return Book' button, users can then click on the folder icon to view the list of books that have been returned. The return list will display the sequence, book code, book title, and author's name.

    When users click 'Save' after returning books, a summary window will appear displaying the returned book details. The summary will include the book code, book title, author's name, borrowing date, and return date.

    4. Book management screen

    4.1 Main screen

    When users access the book management screen, they will find a list of all books available for editing or deletion. Users can enter the book's name they wish to find, then press the search button. The screen will then display the list of books matching the user's search criteria.

    Furthermore, on the book management page, there are buttons for adding new books, adding old books, and checking book conditions, allowing users to add books and inspect their conditions.

    4.2 Add New Book screen

    When new books arrive at the library, librarians must add them through this screen by entering various details about the books.

    4.3 Add Old Book screen

    In cases where there are donations of old books to the library, librarians will add the books through this screen by entering various details about the books.

    4.4 Book inspection screen

    When librarians access the book inspection screen, they will encounter this page. Librarians can search a book by book code.

    After librarians have selected the book to inspect, if they determine that the book is not damaged, they can press 'Save' to update the status of the book from borrowed to inspected.

    If the librarian finds that the book is damaged or defective, they should select the option for 'damaged book' and enter the fine amount to record the fine information.

    5. Student data management screen

    When accessing the student data management screen, users need to enter the student ID they wish to manage. Then, they should click the search button to display the information of the selected student.

    When entering the student ID, the screen will display the student's personal information, which the user can edit by clicking the 'Edit' button. Additionally, the screen will show a table of all the student's borrowing and returning history for the user to review.

    6. The lost book screen

    When a book is lost, the librarian will change its status to 'lost' by clicking the 'Add' button.

    After clicking the 'Add' button, details of the lost book will appear.

    After clicking the 'Save' button, confirmation page will appear then click confirm button.

    7. Fine payment screen

    When accessing the fine payment page, users need to enter the student ID first. After that, they should click the search button to allow the system to display the information of the desired student.

    After clicking the search button, the screen will display the student's personal information, including their photo, name, grade, class, and student ID. It will also show a table listing all outstanding fines, consisting of columns such as sequence number, book title, borrowing date, return date, fine amount, and details or reasons for each fine, such as late return, damaged book, allowing the user to select fine payment options.

    Users can select payment items by checking the box in the 'Select Payment Item' column. Once they have chosen the items they wish to pay for, they should press the 'Pay Fine' button for the system to summarize the details and total amount due.

    When users press the 'Pay Fine' button, the screen will display detailed information of all fines that need to be paid, allowing users to review before confirming the payment.

    When the user confirms the payment by pressing the 'Confirm Payment' button, the screen will display the payment receipt. Users can then press the 'Print' button to print the receipt. If they do not wish to print, they can press the 'Close' button, and the screen will return to the original fine payment page.

    The report generation screen will display a calendar box for users to select the desired date range for generating the report. After selecting the dates, users can press the 'Confirm' button, and the screen will show the report for the selected date range.
